# Project Overview

## プロジェクト概要
- Music Macro Language (MML) を解釈し、デジタルオーディオとして音楽を再生するプロジェクトです。
- 主にコマンドラインインターフェース (CLI) ツールとして機能し、MML形式の楽譜を扱います。
- Web Audio APIを活用し、ブラウザベースで音楽再生が可能なMMLプレイヤーの構築を目指しています。

## 技術スタック
- フロントエンド: Tone.js - Web Audio APIを抽象化し、ブラウザでの音声合成やエフェクト処理を容易にするJavaScriptライブラリです。Web Audio API - モダンなウェブブラウザに組み込まれている、高度な音声処理を可能にするAPIです。

## ファイル詳細説明
- **`.editorconfig`**: 異なる開発環境間でのコードフォーマットの一貫性を保つための設定ファイルです。
- **`.gitignore`**: Gitがバージョン管理の対象としないファイルやディレクトリを指定するファイルです。
- **`.vscode/`**: Visual Studio Codeエディタのワークスペース固有の設定や推奨拡張機能が格納されているディレクトリです。
    - **`extensions.json`**: プロジェクト開発に推奨されるVS Code拡張機能のリストです。
    - **`settings.json`**: VS Codeワークスペースの動作や表示に関する設定が記述されています。
- **`Cargo.lock`**: Rustプロジェクトの依存関係の正確なバージョンを記録し、再現可能なビルドを保証するためのロックファイルです。
- **`Cargo.toml`**: Rustプロジェクトのマニフェストファイルで、プロジェクト名、バージョン、依存関係、ビルド設定などが定義されています。
- **`IMPLEMENTATION_PLAN.md`**: プロジェクトの具体的な実装計画や手順が記述されたドキュメントです。
- **`LICENSE`**: プロジェクトのライセンス情報が記載されています。
- **`PHASE2_INVESTIGATION_REPORT.md`**: プロジェクトの第2フェーズにおける調査結果や分析がまとめられたレポートです。
- **`PHASE2_STATUS.md`**: プロジェクトの第2フェーズの現在の進捗状況やステータスが記述されたドキュメントです。
- **`README.ja.md`**: プロジェクトの概要、使い方、開発方法などを日本語で説明する主要なドキュメントです。
- **`README.md`**: プロジェクトの概要、使い方、開発方法などを英語で説明する主要なドキュメントです。
- **`_config.yml`**: GitHub Pagesでウェブサイトを公開する際に使用されるJekyllの設定ファイルです。
- **`generated-docs/`**: 自動生成されたドキュメントが格納されるディレクトリです（例: 開発状況レポート）。
- **`issue-notes/`**: プロジェクトの課題（Issue）に関するメモや詳細情報が格納されているディレクトリです。
- **`src/`**: プロジェクトのソースコードが格納されているディレクトリです。
    - **`main.rs`**: Rustアプリケーションのエントリポイントとなるファイルで、MMLパーサーおよびプレイヤーの主要なロジックが実装されています。
