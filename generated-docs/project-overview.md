# Project Overview

## プロジェクト概要
- Music Macro Language (MML) で記述された音楽を再生するWindows用CLIツールです。
- コマンドラインにMML文字列を入力するだけで、シンプルかつ低レイテンシで音楽を演奏します。
- バックグラウンドでサーバーを起動し、複数の演奏リクエストに対応できる機能を持ちます。

## 技術スタック
- フロントエンド: 該当なし (CLIツールとして直接的なフロントエンド技術は使用していません)
- 音楽・オーディオ:
    - **Nuked-OPM**: FM音源チップYM2151エミュレーションライブラリ。
    - **cpal (Rust)**: クロスプラットフォームのオーディオ入出力ライブラリ。音声波形の生成と出力に使用されます。
    - **MML (Music Macro Language)**: ユーザーが音楽を記述するためのテキストベースの言語。
- 開発ツール:
    - **Rust**: 高性能で安全なプログラミング言語。プロジェクトの主要な実装言語です。
    - **Zig cc**: クロスコンパイルをサポートするC/C++コンパイラ。Rustプロジェクトの依存関係をビルドするために使用されます。
    - **Windows**: 主要な開発および実行環境です。
    - **Linux runner**: 開発エージェントのテスト駆動開発 (TDD) のために使用される環境。
    - **ALSA SDK**: Linux環境でのオーディオ関連のTDDをヘッドレス環境で可能にするためのSDK。
- テスト:
    - **Linux runner (agent TDD用)**: 開発エージェントがテスト駆動開発を行うための実行環境。
    - **ALSA SDK**: ヘッドレスト環境でのオーディオ関連のテストを可能にするためのSDK。
- ビルドツール:
    - **Cargo**: Rustプロジェクトのビルドシステムおよびパッケージマネージャー。
- 言語機能:
    - **Rust**: 安全性、並行性、パフォーマンスに重点を置いたモダンなプログラミング言語。
    - **MML (Music Macro Language)**: 音符、音長、テンポ、オクターブなどをテキストで表現する記法。
- 自動化・CI/CD:
    - **GitHub Actions**: READMEの日本語から英語への自動翻訳に使用されています。
- 開発標準:
    - **.editorconfig**: 異なるエディタ間でのコーディングスタイルの一貫性を保つための設定ファイル。
    - **.gitignore**: Gitバージョン管理システムが無視すべきファイルを指定します。
    - **.vscode/settings.json**: Visual Studio Codeのワークスペース設定を定義し、プロジェクト固有の開発環境を標準化します。

## ファイル詳細説明
- **`.editorconfig`**: コードエディタ間でインデントスタイルや文字コードなどの基本的な書式設定を統一するための設定ファイルです。
- **`.gitignore`**: Gitがバージョン管理の対象から除外するファイルやディレクトリを指定するファイルです。ビルド生成物や一時ファイルなどが含まれます。
- **`.vscode/extensions.json`**: Visual Studio Codeユーザー向けに、プロジェクト推奨の拡張機能をリストアップするファイルです。
- **`.vscode/settings.json`**: Visual Studio Codeのワークスペース固有の設定を定義するファイルです。プロジェクトごとに異なるエディタ設定を適用できます。
- **`Cargo.lock`**: Rustプロジェクトの依存関係ツリーと、その正確なバージョンを記録するファイルです。これにより、ビルドの一貫性が保証されます。
- **`Cargo.toml`**: Rustプロジェクトのマニフェストファイルです。プロジェクト名、バージョン、著者、依存クレートなどが記述されています。
- **`LICENSE`**: プロジェクトのライセンス情報（MIT License）が記載されています。
- **`README.ja.md`**: プロジェクトの目的、使い方、技術詳細などを日本語で説明するファイルです。
- **`README.md`**: プロジェクトの目的、使い方、技術詳細などを英語で説明するファイルです。`README.ja.md`から自動生成されています。
- **`_config.yml`**: GitHub Pagesなどのサイト生成ツールで使用される設定ファイルである可能性があります。
- **`generated-docs/`**: プロジェクトのドキュメントが自動生成されて格納されるディレクトリです。
- **`src/`**: プロジェクトのソースコードを格納するディレクトリです。
    - **`src/app.rs`**: アプリケーションの主要なロジックや、MMLの解析結果からオーディオを生成・再生するコア部分を定義している可能性があります。
    - **`src/cli.rs`**: コマンドラインインターフェース（CLI）の引数解析、コマンドの処理、ヘルプ表示などの機能を提供します。
    - **`src/client_manager.rs`**: CLIツールがサーバーモードで動作している場合に、クライアントからのリクエスト（MML文字列や制御コマンド）を管理する役割を担います。
    - **`src/converter.rs`**: MMLテキストをプログラムが扱える中間表現（ASTや音楽データ構造）に変換するパーサーと関連ロジックを実装しています。
    - **`src/input.rs`**: ユーザーからのMML入力文字列を受け取り、初期処理を行う部分に関連する可能性があります。
    - **`src/main.rs`**: プログラムのエントリポイントであり、アプリケーションの起動、CLI引数の解釈、適切な処理の呼び出しを行います。
    - **`src/process_manager.rs`**: バックグラウンドで動作するサーバープロセス（オーディオ再生プロセス）の起動、停止、シャットダウンなどのライフサイクルを管理します。
